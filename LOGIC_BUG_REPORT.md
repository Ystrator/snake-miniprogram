# 育儿小程序功能逻辑Bug报告

## 发现的严重逻辑Bug

### Bug #1: 文章分类字段不匹配 🔴 严重
**位置**: `data.js` 与 `category.js`, `index.js`, `search.js`
**问题**: 
- 数据文件中的文章使用 `category` 字段（字符串值，如"0-6个月"）
- 代码逻辑中使用 `categoryId` 字段（分类ID，如"age-0-1"）
- 导致分类筛选功能完全失效

**影响范围**:
- 首页推荐文章功能失效
- 分类页面无法正确筛选文章
- 搜索功能的分类筛选失效

**示例**:
```javascript
// data.js 中的文章数据
{
  "id": "article_001",
  "category": "0-6个月",  // ❌ 使用的是字符串
  ...
}

// 但代码中期望
article.categoryId === 'age-0-1'  // ❌ 永远不会匹配
```

---

### Bug #2: 年龄计算逻辑错误 🟡 中等
**位置**: `app.js` 的 `calculateBabyAge()` 方法
**问题**:
- 使用了两种不同的月龄计算方式（`months` 和 `ageMonths`）
- 返回的 `totalMonths` 基于日历月份计算，可能不准确
- 推荐文章时使用 `totalMonths`，但显示时使用 `months` 和 `days`

**影响**:
- 年龄显示可能不准确
- 推荐文章分类可能选择错误

**当前代码问题**:
```javascript
// 计算方式1：基于日历月份
let months = (today.getFullYear() - birthDate.getFullYear()) * 12;
months -= birthDate.getMonth();
months += today.getMonth();

// 计算方式2：基于天数
const days = Math.floor((today - birthDate) / (1000 * 60 * 60 * 24));
const ageMonths = Math.floor(days / 30);
const ageDays = days % 30;

// 返回值混乱
return {
  totalMonths: months,  // 基于日历
  months: ageMonths,    // 基于天数
  days: ageDays
};
```

---

### Bug #3: 搜索功能的相关度算法不完善 🟡 中等
**位置**: `search.js` 的 `doSearch()` 方法
**问题**:
- 搜索时没有检查 `description` 字段是否为 `undefined`
- `summary` 和 `description` 可能同时存在，导致重复计算
- 相关度评分算法简单，可能不够准确

**影响**:
- 搜索结果排序可能不准确
- 可能出现 undefined 错误

---

### Bug #4: 宝宝信息保存时缺少数据验证 🟡 中等
**位置**: `baby.js` 的 `confirmDate()` 方法
**问题**:
- 没有验证日期是否在未来（不能选择未来日期作为生日）
- 没有验证日期是否过于久远（如100年前）
- 名字输入后没有及时更新到本地存储

**影响**:
- 可能保存不合理的生日数据
- 用户体验不佳

---

### Bug #5: 首页推荐文章逻辑存在空数据风险 🟠 轻微
**位置**: `index.js` 的 `filterRecommendedArticles()` 方法
**问题**:
- 当某个分类下没有文章时，`hotArticles` 会是空数组
- 没有降级策略（如显示其他分类文章）

**影响**:
- 用户可能看到空白区域

---

### Bug #6: 收藏功能ID不一致 🟠 轻微
**位置**: `article.js`
**问题**:
- 检查收藏状态时使用 `articleId`
- 但文章ID可能是 `article_001` 格式或 `001` 格式
- 导致收藏状态判断错误

**影响**:
- 收藏按钮状态可能显示错误

---

### P0 - 立即修复（影响核心功能）
1. ✅ **Bug #1**: 文章分类字段不匹配 - 统一字段名称

### P1 - 高优先级（影响用户体验）
2. ✅ **Bug #2**: 年龄计算逻辑错误 - 统一计算方式

### P2 - 中优先级（功能增强）

5. ✅ **Bug #5**: 首页推荐空数据处理

### P3 - 低优先级（小问题）

## 修复方案概览

### 1. 数据结构统一
在 `data.js` 中为每篇文章添加正确的 `categoryId` 字段

### 2. 年龄计算优化
统一使用基于天数的月龄计算方式

### 3. 搜索算法增强
改进相关度评分，添加字段验证

### 4. 数据验证
添加日期范围验证和空数据检查

### 5. 降级策略
添加推荐文章为空时的后备方案

---
