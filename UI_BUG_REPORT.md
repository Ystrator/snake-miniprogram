## 发现的 UI Bug

### 🔴 严重问题

#### 1. 数据结构不匹配导致首页文章无法显示
**位置**: `pages/index/index.js` - `loadArticles()` 方法

**问题描述**:
- data.js 中的文章使用 `content` 数组结构
- article.wxml 期望的是 `sections` 数组结构
- 导致文章详情页面内容无法正常渲染

**影响**: 用户点击文章后详情页面空白或内容不完整

**修复方案**: 
需要统一数据结构或添加数据转换层

---

#### 2. 搜索页面使用了不存在的 `knowledgeData.allArticles`
**位置**: `pages/search/search.js` - `doSearch()` 方法

**问题描述**:
```javascript
let allArticles = knowledgeData.allArticles.map(...)
```
但 data.js 中的结构是:
```javascript
{
  categories: [...],
  allArticles: [...]
}
```

**影响**: 搜索功能完全失效

**修复方案**:
修改搜索逻辑，从 categories 中提取所有文章

---

#### 3. 文章详情页硬编码文章数据
**位置**: `pages/article/article.js`

**问题描述**:
- 文章数据硬编码在 article.js 中
- 与 data.js 中的数据不同步
- 导致只能查看12篇固定文章

**影响**: 
- 用户无法查看data.js中的80篇文章
- 新增文章无法显示

**修复方案**:
从 data.js 动态加载文章数据

---

### 🟡 中等问题

#### 4. 搜索结果文章ID格式不匹配
**位置**: `pages/search/search.js`

**问题描述**:
- data.js 中的文章ID格式: `article_001`, `article_002`
- article.js 中期望的ID格式: `001`, `002`
- 导致搜索结果点击后无法打开文章

**影响**: 搜索功能无法正常跳转到文章详情

**修复方案**:
统一文章ID格式或在跳转时进行转换

---

#### 5. 分类页面硬编码文章列表
**位置**: `pages/category/category.js`

**问题描述**:
- 分类页面文章列表硬编码
- 与 data.js 不同步

**影响**: 分类显示的文章数量受限

**修复方案**:
从 data.js 动态加载分类文章

---

### 🟢 轻微问题

#### 6. CSS变量在dark-mode下的兼容性
**位置**: `app.wxss`

**问题描述**:
部分样式使用了CSS变量，但在夜间模式切换时可能存在延迟

**影响**: 夜间模式切换时可能出现闪烁

**优化建议**:
添加过渡动画平滑切换

---

#### 7. 文章详情页快速解答板块样式在夜间模式下对比度不足
**位置**: `pages/article/article.wxss`

**问题描述**:
`.quick-answers-box` 在夜间模式下缺少特定样式

**影响**: 夜间模式下快速解答板块可读性降低

**修复方案**:
添加夜间模式专用样式

## 总结

当前小程序存在7个UI相关bug，主要集中在：
1. **数据结构不统一** - 不同页面使用不同的数据格式
2. **硬编码数据** - 多个页面使用硬编码数据而非动态加载
3. **ID格式不一致** - 导致页面跳转失败

建议优先修复P0级别问题，确保基本功能可用。
