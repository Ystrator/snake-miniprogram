## 问题诊断

### 1. 分类字段缺失问题 ✅ 已修复
**问题描述:**
- data.js 中的文章使用 `"category": "0-6个月"` (文本值)
- 代码期望使用 `"categoryId": "age-0-1"` (ID值)
- 导致分类筛选功能失效

**修复方案:**
- 为所有80篇文章添加 `categoryId` 字段
- 建立分类映射关系:
  - "0-6个月" → "age-0-1"
  - "6-12个月" → "age-0-1"
  - "1-3岁" → "age-1-3"
  - "3-6岁" → "age-3-6"
  - "通用知识" → "general"

**修复结果:**
```
✓ 80篇文章全部添加 categoryId
✓ JSON语法验证通过
✓ 分类分布:
  - age-0-1: 45篇
  - age-1-3: 19篇
  - age-3-6: 5篇
  - general: 11篇
```

### 2. 文章ID格式问题 ✅ 已修复
**问题描述:**
- data.js 中使用 `"article_001"`, `"article_002"` 等格式
- article.js 中硬编码的文章使用 `"001"`, `"002"` 等格式
- 导致文章详情页无法找到文章

**修复方案:**
- 修改 article.js，使其从 data.js 动态加载文章
- 支持多种ID格式的兼容性查找:
  - 直接匹配: `article_001`
  - 去前缀匹配: `001` → `article_001`
  - 加前缀匹配: `001` → `article_001`

**修复结果:**
```
✓ 文章详情页可以正确加载所有80篇文章
✓ 支持向后兼容多种ID格式
```
