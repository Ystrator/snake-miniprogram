## 🔴 发现的Bug清单

### Bug #1: 数据导入错误（严重）
**文件:** `pages/index/index.js`
**位置:** 第73行
**问题:** 错误的导入语法
```javascript
// 错误
const knowledgeData = require('../../data.js').knowledgeData;

// 正确
const knowledgeData = require('../../data.js');
```
**影响:** 导致首页无法加载数据，文章列表为空
**状态:** ✅ 已修复

---

### Bug #2: 数据结构不匹配（严重）
**文件:** `data.js` + 所有页面
**问题:**
1. `data.js`中的`categories`数组的`articles`字段为空
2. 实际的80篇文章存储在`allArticles`数组中
3. 文章使用`category`字段（字符串），而不是`categoryId`

**影响:** 所有页面无法正确加载和显示文章
**状态:** ✅ 已修复

**修复方案:**
- 首页：添加`category`→`categoryId`映射逻辑
- 搜索页：添加分类映射，从`allArticles`读取数据
- 分类页：根据`category`字段筛选文章
- 文章页：支持`article_XXX`和`XXX`两种ID格式

---

### Bug #3: 分类页面硬编码数据（中等）
**文件:** `pages/category/category.js`
**问题:** 分类页面使用硬编码的12篇示例文章，未使用真实数据
**影响:** 无法显示完整的80篇文章
**状态:** ✅ 已修复

**修复内容:**
- 移除硬编码数据
- 从`data.js`加载真实文章
- 根据分类筛选文章（支持`0-6个月`, `6-12个月`等分类）

---

### Bug #4: 文章ID格式不一致（中等）
**文件:** `pages/article/article.js`
**问题:**
- `data.js`中文章ID格式为`article_001`, `article_002`
- 旧代码期望`001`, `002`格式
- 导致文章详情页无法找到文章

**影响:** 点击文章无法查看详情
**状态:** ✅ 已修复

**修复内容:**
- 支持两种ID格式（`article_XXX`和`XXX`）
- 从`data.js`加载真实文章数据
- 添加分类映射逻辑

---

### Bug #5: 搜索功能数据源错误（中等）
**文件:** `pages/search/search.js`
**问题:** 尝试从空的`categories[i].articles`读取文章
**影响:** 搜索功能无结果
**状态:** ✅ 已修复
